The flex attribute (flex-grow, flex-shrink, flex-basis)

These are properties that are applied directly to the children; they say how much space they take.

flex: 1 2 auto; is the same as flex-grow: 1; flex-shrink: 2; flex-basis: auto

With the defaults

With flex: 0 1 auto, the elements take only the space needed by their content. This is the default (same as not writing it).

With flex-basis: 0

flex-basis: 0 means: assume the content has a width of 0 (like there was no content). Items will compress their content as much as possible.

With flex-grow: 1

flex-basis: auto; flex-grow: 1 means: start with the space needed by the content, then divide the available space equally between all items

flex: 1 1 auto
A longer item, but flex: 1 1 auto

With different flex-grow values

The flex-grow values of all children are summed; then, a proportion is calculated for each one. The available space is divided with that proportion.

Here, the total is 1+2+1 = 4. The center item ends up with 2/4 = 50% and the others end up with 1/4 = 25% of the available space.

With flex-shrink: 0

With flex-shrink: 0, elements are not allowed to shrink to fit the available space (by default, they will).

A longer item, but flex: 0 0 auto

With flex-shrink: 1 (default)

When needed, by default elements will shrink in a proportion adequate to their content.

The elements take the minimum space for their contents (for example, breaking lines between words).

Elements with more content get more space removed, even if flex-shrink is the same in all elements.

With different flex-shrink values

Higher values = more space gets taken from the element.

This is also a proportion, similar to flex-grow. Here, the shrink total is 1+4+1+1 = 7; the 2nd element is shrunk more (4/7 = 0.57) and the rest, less (1/7 = 0.14)
